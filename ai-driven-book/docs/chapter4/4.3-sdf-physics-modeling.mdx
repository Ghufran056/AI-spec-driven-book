# SDF & Physics Modeling

## Learning Objectives

By the end of this lesson, you will be able to:
- Understand the purpose and structure of SDF in robot modeling
- Distinguish between SDF and URDF for simulation applications
- Create SDF files for physics simulation in Gazebo
- Define physics properties and material properties in SDF
- Configure collision and friction parameters for realistic simulation
- Apply SDF concepts to model complex robot-environment interactions

## Introduction

SDF (Simulation Description Format) is an XML-based format specifically designed for physics simulation in environments like Gazebo. While URDF is primarily used for robot kinematics and basic geometric representation, SDF extends this to include detailed physics properties, sensor models, and environment descriptions necessary for realistic simulation. SDF files provide the information needed by physics engines to accurately simulate the behavior of robots and their interactions with the environment, including collision detection, friction, damping, and other physical phenomena.

SDF serves as the native format for Gazebo and other simulation environments, providing a comprehensive description of the physical world including robots, objects, lighting, and environmental conditions. Unlike URDF which focuses on robot structure, SDF encompasses the entire simulation world with detailed physics properties. SDF can include URDF models as components, allowing for hybrid approaches that combine the kinematic descriptions of URDF with the physics extensions of SDF. This makes SDF particularly valuable for creating realistic simulation scenarios that accurately reproduce the physical behaviors expected in the real world.

## Main Theory

### 1. SDF vs URDF
While URDF focuses on robot kinematics and basic geometry, SDF extends this to include detailed physics properties, sensor models, and environment descriptions. SDF can include URDF models as components but provides additional physics-specific features.

### 2. SDF World Structure
SDF files can describe entire worlds including models, lights, physics engines, and plugins. The world element contains multiple model elements, each representing a physical object in the simulation.

### 3. Physics Properties
SDF allows detailed specification of physical properties including mass, moments of inertia, friction coefficients, restitution coefficients, and damping parameters that affect how objects behave in simulation.

### 4. Collision Properties
SDF provides more detailed collision modeling than URDF, including surface properties like friction, contact models, and custom collision shapes that can be optimized for physics simulation.

### 5. Sensor Models
SDF includes native support for various sensor types (cameras, LIDAR, IMU, etc.) with realistic noise models and parameters that affect their behavior in simulation.

### 6. Plugins
SDF supports plugins that extend simulation capabilities, allowing for custom controllers, sensors, and other functionality to be integrated directly into the simulation environment.

```
SDF World Structure:

<sdf version='1.7'>
  <world name='default'>
    <physics type='ode'>
      <gravity>0 0 -9.8</gravity>
      <max_step_size>0.001</max_step_size>
    </physics>

    <light name='sun' type='directional'>
      <pose>0 0 10 0 0 0</pose>
    </light>

    <model name='ground_plane'>
      <static>true</static>
      <link name='link'>
        <collision name='collision'>
          <geometry>
            <plane><normal>0 0 1</normal></plane>
          </geometry>
        </collision>
      </link>
    </model>

    <model name='robot' canonical='base_link'>
      <!-- Robot model definition -->
    </model>
  </world>
</sdf>
```

## Examples

### Example: Simple SDF World with Physics
```xml
<?xml version="1.0" ?>
<sdf version="1.7">
  <world name="simple_world">
    <!-- Physics engine configuration -->
    <physics name="default_physics" type="ode">
      <gravity>0 0 -9.8</gravity>
      <max_step_size>0.001</max_step_size>
      <real_time_factor>1</real_time_factor>
      <real_time_update_rate>1000</real_time_update_rate>
    </physics>

    <!-- Ground plane -->
    <model name="ground_plane">
      <static>true</static>
      <link name="link">
        <collision name="collision">
          <geometry>
            <plane><normal>0 0 1</normal></plane>
          </geometry>
          <surface>
            <friction>
              <ode>
                <mu>1.0</mu>
                <mu2>1.0</mu2>
              </ode>
            </friction>
          </surface>
        </collision>
        <visual name="visual">
          <geometry>
            <plane><normal>0 0 1</normal><size>100 100</size></plane>
          </geometry>
          <material>
            <ambient>0.8 0.8 0.8 1</ambient>
            <diffuse>0.8 0.8 0.8 1</diffuse>
          </material>
        </visual>
      </link>
    </model>

    <!-- Simple box model -->
    <model name="box">
      <pose>0 0 1 0 0 0</pose>
      <link name="link">
        <inertial>
          <mass>1.0</mass>
          <inertia>
            <ixx>0.1667</ixx>
            <ixy>0</ixy>
            <ixz>0</ixz>
            <iyy>0.1667</iyy>
            <iyz>0</iyz>
            <izz>0.1667</izz>
          </inertia>
        </inertial>
        <collision name="collision">
          <geometry>
            <box><size>1 1 1</size></box>
          </geometry>
          <surface>
            <friction>
              <ode>
                <mu>0.5</mu>
                <mu2>0.5</mu2>
              </ode>
            </friction>
            <bounce>
              <restitution_coefficient>0.1</restitution_coefficient>
            </bounce>
          </surface>
        </collision>
        <visual name="visual">
          <geometry>
            <box><size>1 1 1</size></box>
          </geometry>
          <material>
            <ambient>0.5 0 0 1</ambient>
            <diffuse>1 0 0 1</diffuse>
          </material>
        </visual>
      </link>
    </model>
  </world>
</sdf>
```

## Practical Notes

- SDF is the native format for Gazebo simulation environment
- Use SDF for detailed physics properties and sensor models
- SDF files can include URDF models using the `<include>` tag
- Physics parameters like friction and restitution greatly affect simulation realism
- Use appropriate solver parameters for stable simulation
- Test physics parameters against real-world behavior when possible
- SDF supports plugins for custom simulation behaviors

## Summary

SDF is a powerful format for describing physics simulation environments, extending beyond basic robot kinematics to include detailed physical properties, sensors, and environmental conditions. While URDF handles robot kinematic descriptions, SDF provides the physics properties necessary for realistic simulation in environments like Gazebo. Understanding both formats and their appropriate use cases is essential for creating effective robot simulation environments that accurately reproduce real-world behaviors.

## Glossary

- **SDF**: Simulation Description Format - XML-based format for physics simulation
- **Physics Engine**: Software component that calculates realistic physical interactions in simulation
- **Gravity**: Global acceleration applied to all objects in the simulation
- **Max Step Size**: Maximum time step for physics integration (smaller = more accurate)
- **Real Time Factor**: Ratio of simulation time to real time
- **Collision Properties**: Parameters that define how objects interact when they collide
- **Friction**: Force that resists sliding motion between contacting surfaces
- **Restitution**: Measure of elasticity in collisions (bounciness)
- **Damping**: Force that opposes motion to reduce oscillation
- **Surface Properties**: Material properties that affect contact behavior
- **ODE**: Open Dynamics Engine - physics simulation engine
- **Contact Model**: Method for calculating forces when objects touch
- **Inertial Properties**: Mass, center of mass, and moment of inertia parameters
- **Plugin**: Extension code that adds functionality to the simulation
